💡 POINTS CLÉS POUR VOTRE TP
═════════════════════════════════════════════════════════════════════════════════

1. Le logging est COMPLÈTEMENT AUTOMATIQUE
   → Aucun code côté client nécessaire
   → Chaque GET est loggé automatiquement

2. L'IP du client est extraite automatiquement
   → Depuis les headers x-forwarded-for (Azure)
   → Ou remoteAddress si pas de proxy

3. Les logs sont en MÉMOIRE
   → Idéal pour le TP
   → Pour la production : utilisez une base de données

4. L'interface est RESPONSIVE
   → Fonctionne sur mobile + desktop
   → Rafraîchissement automatique

5. Tout est prêt pour Azure App Service
   → Configuration IIS incluse
   → Déploiement automatisé


🎓 UTILISATION POUR LE TP
═════════════════════════════════════════════════════════════════════════════════

ÉTAPE 1 : Déployer
    .\deploy.ps1

ÉTAPE 2 : Attendre (1-2 min)
    Laissez Azure déployer l'application

ÉTAPE 3 : Accéder
    Ouvrez https://webapp-cloud-lab.azurewebsites.net

ÉTAPE 4 : Générer des logs
    • Visitez différentes URL (/path1, /path2, etc.)
    • Utilisez depuis différents appareils/réseaux
    • Cliquez sur "Requête de test"

ÉTAPE 5 : Observer
    • Voir les IPs des visiteurs
    • Voir les chemins visités
    • Voir les statistiques en temps réel

ÉTAPE 6 : Analyser
    • Combien d'IPs uniques ?
    • Quels chemins sont les plus visités ?
    • Quels User-Agents sont présents ?
